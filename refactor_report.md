# Отчёт о рефакторинге

## Общие изменения
- Пакет `graphics` переименован в `client`, чтобы отделить клиентскую часть и подготовить проект к разделению клиент/сервер; все графические классы перенесены в `SwordsGame.client`.
- Блоки вынесены в `SwordsGame.client.blocks`: добавлены отдельные классы блоков и единый реестр, что упрощает расширение набора блоков.
- Серверная логика разбита по назначению: генерация в `SwordsGame.server.structures`, функциональные действия в `SwordsGame.server.functions`.
- Генератор террейна перенесён в `SwordsGame.server.Terrain`, а пакет `server.structures` сохранён для будущих структур.
- Добавлен единый набор констант путей текстур в `SwordsGame.client.assets.Paths` для унификации строк ресурсов и вынесен отдельный режим `Debug`.
- Переименованы сущности блоков и реестра (`Registry`, `Type`, `Stone`, `Grass`, `Cobble`) для более чистых названий внутри пакета `blocks`.
- В окне добавлены ограничения размеров для фиксации виртуального разрешения.

## Детальные изменения
- `Block.draw`: таблица UV вынесена в статическую константу, чтобы избежать пересоздания массива и упростить отрисовку.
- `World.renderChunkCached`: проверка видимости граней вынесена в отдельный метод `isAnyFaceVisible`, логика стала более читаемой.
- `Text.draw` и `Sprite.draw`: логика вычисления якоря вынесена в `buildAnchor`, устранены дублирующиеся вычисления.
- `World.renderChunkBounds`: добавлена отрисовка границ чанков, вызываемая только из `Debug`, и смещена на половину блока для корректного совпадения с границами блоков.
- Удалены устаревшие/общие комментарии по всему проекту; сохранены только необходимые краткие описания через структуру кода.
